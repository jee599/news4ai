### 3. 경쟁 분석

#### 한국어 AI 뉴스레터
- **GeekNews** — 테크 전반, AI 전용 아님
- **AI 타임스** — 미디어, 뉴스레터 아님
- **직접 경쟁자 거의 없음** — 한국어 AI 전문 자동 뉴스레터는 블루오션

#### 영어 AI 뉴스레터
- **TLDR AI** — 50만+ 구독자, 매일 발행, 큐레이션
- **Ben's Bites** — 45만+ 구독자, AI 스타트업 중심
- **The Rundown AI** — 60만+ 구독자, 짧은 요약
- **Alpha Signal** — AI 연구 중심, 기술적
- **차별화 포인트:** 완전 자동화 (인간 큐레이터 없음) + 실시간 breaking alert

### 4. Phase 1 MVP 범위 확정

**만든다:**
- 이메일 구독 폼 (jidonglab.com/ai-news 하단)
- `/api/subscribe` — Redis에 저장 + 확인 이메일
- `/api/unsubscribe` — 즉시 삭제
- Cloudflare Worker — 매일 9AM KST에 전날 뉴스 이메일 발송
- 이메일 HTML 템플릿 (제목 + 요약 + 링크)

**안 만든다 (Phase 1에서):**
- 웹 대시보드 (구독자 통계)
- 구독자 세그먼트 (키워드별 필터)
- 유료 구독
- A/B 테스트
- 오픈율/클릭율 추적 (Resend 대시보드로 충분)

### 8. Phase 2~3 추가 고려사항

#### Phase 2: 크롤러 고도화
- 현재 generate-ai-news.ts는 Claude Sonnet으로 글을 "작성"한다. Phase 2에서는 "수집"과 "작성"을 분리해야 한다.
  - 수집 Worker: 15분마다 RSS/API fetch → Redis에 raw 뉴스 저장
  - 작성 Worker: 1일 2회 → Redis에서 raw 뉴스 꺼내서 Claude로 글 작성
  - 이렇게 하면 이메일 다이제스트는 작성된 글이 아니라 수집된 raw 뉴스로도 보낼 수 있다.

#### Phase 3: 앱 MVP
- 앱 첫 버전은 WebView + push notification만으로 충분할 수 있다.
  - jidonglab.com/ai-news를 WebView로 감싸고
  - push notification 등록만 네이티브로 구현
  - 2-3일이면 앱 스토어 제출 가능
  - 피드백 받으면서 네이티브 화면으로 점진 전환

#### 데이터 파이프라인 통합
```
Phase 2 크롤러
  → raw 뉴스 (Redis)
  → 중요도 판정 (Haiku)
  ├─ score 8+ → Phase 3 push notification
  ├─ score 5+ → Phase 1 이메일 다이제스트
  └─ 전체 → 1일 2회 Claude 글 작성 → jidonglab.com
```

이 구조가 되면 크롤러 1개가 이메일, 앱, 웹사이트 3개 채널을 동시에 먹인다.
